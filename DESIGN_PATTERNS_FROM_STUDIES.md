## 1. 배너 & 타이틀 — "첫인상 3초"

### 1-1. 왕관형 배너 (Top Banner) 패턴
레퍼런스에서 관찰:
- **타이틀 영역은 화면 상단 10~15% 높이**로 고정
- 좌우 마진 없이 **전폭** 가로지르되, **중앙 왕관 문양** 이 상징 축
- 색상: 다크 석재 배경 + **앰버/금** 악센트
- 텍스트: 세리프 고딕 폰트, 대문자 스타일

**RoF 적용**:
- `.screen` 최상단에 banner 영역을 `<header>` 성격으로 고정
- 배너 내부: 좌측 작은 문장(화면 종류 아이콘) + 중앙 타이틀 텍스트 + 우측 골드/HP HUD
- 세리프는 이미 `var(--font-title)` 사용 중 → 배너 타이틀에 강제

### 1-2. 글로우 & 맥동 애니메이션
- 타이틀 로고는 **정적 절대 금지** — 약한 glow 맥동 (3초 주기)
- 전설/신 등급 배너는 **금빛 스파클 파티클**
- 전투 결과 배너 (승리/패배): 화면 풀스크린 덮고 1.5초 페이드인

**RoF 현황**: `@keyframes titleGlow` 이미 적용 ✅. 배너 glow 맥동은 모든 주요 화면 제목에 확장 검토.

### 1-3. 배너 안의 정보 밀도
- 타이틀 배너에는 **최대 3종 정보**만 배치: (아이콘 · 타이틀 · 자원 1개)
- 그 이상이면 HUD 분리
- 정보 4개째는 반드시 별도 서브 패널로

**RoF 적용 포인트**: 현재 HUD 가 과도한 정보 (골드/명예/리그/연승/일자) → 분산 필요.

## 2. 카드 프레임 — "유닛 정체성의 그릇"

### 2-1. 프레임은 "5등급 스탬프" 역할
- 등급(일반/희귀/전설/신화 등)을 **프레임 자체로 시그널**
- 플레이어는 0.2초 안에 등급 식별해야 함
- 테두리 두께·색·글로우 강도 3단계 변화로 차등화

**RoF 현황**: 5프레임(bronze/silver/gold/legendary/divine) 별 PNG 사용 중 ✅.
**개선점**: 현재는 PNG 교체 방식. 드롭-쉐도우 + 힐라이트 애니메이션 추가 고려.

### 2-2. 카드 정보 계층 (Z축)
레퍼런스 게임들의 공통 패턴:
```
z=1: 일러스트 (배경)
z=2: 프레임 PNG (중간)
z=3: 숫자 슬롯 (HP/ATK 등, 프레임 홈에 정렬)
z=4: 상태 뱃지 (버프/디버프, 임시)
z=5: 선택 오라 / 호버 글로우
```

**RoF 현황**: CardComponent 이미 이 구조 ✅.

### 2-3. "한 장 = 숫자 2~3개만" 원칙
- 카드 위 보이는 숫자가 **3개 초과** 시 인지 과부하
- 나머지는 롱프레스/호버로 **탭 후 공개**
- 전장 위에서는 **HP + ATK 2개만** (나머지 숨김)

**RoF 현황**: 전장 미니 타일은 2개만 ✅.
**개선점**: 컬렉션/선술집 카드도 풀 스탯 대신 **핵심 3개 + 호버 전체** 고려.

### 2-4. 능력 텍스트는 "키워드 + 수치"
- 긴 설명문 ❌ → "도발. 피격 시 공격자에게 3 피해" 형식
- 키워드는 **굵게 + 색 강조**
- 수치는 **크고 명확한 색**(주황/빨강)

**RoF 적용**: 판타지 몰입형 텍스트 규칙과 충돌. 해법:
- **카드 앞면**: 키워드 요약 (ex: "**도발**. 먼저 공격 받음")
- **롱프레스 상세**: 몰입형 문장 ("이 자는 동료들의 방패가 되어 적의 시선을 끈다...")

## 3. 마을/맵 화면 — "여정의 홈"

### 3-1. 마을은 "선택의 조망대"
레퍼런스 공통 패턴:
- 하나의 **일러스트 배경** (쿼터뷰 or 이소메트릭)
- 그 위에 **클릭 가능 핫스팟** 5~10개
- 각 핫스팟은 호버 시 **글로우 + 이름 라벨**
- 클릭 시 해당 건물로 **줌인 전환** (카메라 이동 느낌)

**RoF 현황**: 848×1264 세로 맵 + 8개 빌딩 배치 ✅.
**개선점**:
- [ ] 건물 호버 시 **앰버 글로우** + 이름 라벨 (현재 부족)
- [ ] 클릭 시 **줌인 카메라 애니메이션** (현재 즉시 전환)
- [ ] 배경에 **유휴 애니메이션**: 깃발 흔들림, 연기 피어오름, 횃불 깜빡임 (design HANDOFF 후속 계획에 이미 있음)

### 3-2. 여정의 "진척 시각화"
- 레퍼런스에서 관찰: 방문한 건물 / 업그레이드한 건물이 **시각적으로 다름**
- 건물 렌더가 **레벨에 따라 발달** (오두막 → 성채)
- 도로/강이 **여정의 흐름** 표현

**RoF 현황**: 레벨 시스템 있음 ✅ (Lv1~5). 레벨별 건물 PNG 예정.
**개선점**: 마을 배경 자체가 **티어 1/2/3** 으로 발달하면 성장감 ↑ (design HANDOFF 후속 2번)

### 3-3. "대화 NPC" 서사 보강
- 각 건물 진입 시 NPC 가 **레벨별 다른 인사말**
- 빌딩 업그레이드마다 **NPC 도 교체**
- NPC 대사는 **플레이어 레벨/진척에 반응**

**RoF 현황**: `NPCS` 배열에 이미 레벨별 5명 NPC 정의됨 ✅.

## 4. HUD & 자원 표시 — "항상 보이는 3가지"

### 4-1. 상단 HUD 는 "3자원 + 아이덴티티"
- 좌측: 플레이어 아이덴티티 (이름 · 영웅 썸네일)
- 중앙: 현재 자원 (골드/HP/AP 중 2~3)
- 우측: 시간/라운드/페이즈 정보
- **4개 초과하면 공간 부족 + 시각 노이즈**

**RoF 현황**: `.hud` 가 중앙 자원 과잉 → 3개로 압축 필요.

### 4-2. 자원 변동 **플로팅 숫자**
- 골드 +20 → 금빛 "+20" 이 HUD 자원 위로 떠올라 페이드아웃
- HP -5 → 빨간 "-5" 이 하트 위로
- **모든 자원 변화는 시각 피드백** 필수

**RoF 현황**: 일부 있음 (전투). **비전투 화면(선술집/성채)에도** 확장 필요.

### 4-3. 자원 부족 시 **호흡 애니메이션**
- 골드 부족 → 골드 아이콘 **빨간 맥동**
- 덱 가득참 → 덱 카운터 **주황 맥동**
- 플레이어가 "왜 안 되지?" 를 즉각 인지

## 5. 버튼 & 상호작용 — "4상태 필수"

### 5-1. 버튼 4상태
1. **Idle**: 기본
2. **Hover**: 밝기 +15%, 약한 글로우
3. **Active (press)**: scale(0.97), 짙은 그림자 감소
4. **Disabled**: 채도 -60%, 커서 not-allowed, opacity 0.5

**RoF 현황**: `.claude/skills/rof-game-feel.md` 에 이미 상세 스펙 있음 ✅. 모든 `.btn` 에 적용 완료.

### 5-2. 중요 버튼 = **배너형**
- "도전자 찾기" / "확인" / "전투 시작" 같은 결정 버튼은 **크고 화려하게**
- 일반 버튼 대비 **폭 2배, 높이 1.3배**
- 금색 테두리 + 맥동 glow

**RoF 현황**: `.btn-green` `btn-red` 색 구분은 있으나 **크기 차등** 부족.

### 5-3. 버튼 그룹 정렬
- 관련 버튼은 **수평 그룹**, 간격 8~12px
- 주요/보조 버튼은 **위치로 구분**: 주요는 우하단/중앙, 보조는 좌측
- **닫기/취소는 항상 좌측**, **확정/다음은 항상 우측**

---

## 6. 키워드 툴팁 — "복잡도 은닉"

### 6-1. 3단계 계층
1. **카드 텍스트**: 키워드 굵게만 (**도발**)
2. **호버 툴팁**: 1줄 정의 ("먼저 공격 받음")
3. **키워드 창**: 전체 설명 + 관련 카드 목록 (컬렉션)

**RoF 미구현** → PHASE2_KEYWORD_PLAN.md 있음. 실구현 단계에서 이 3계층 구조 강제.

### 6-2. 키워드 이름 규칙
- **2~3음절 한국어** 우선 (도발/선제/관통/흡수/반격)
- 외래어 금지 (퍼스트스트라이크 ❌)
- 3음절 초과 시 2음절로 줄이거나 다른 단어 채택

## 7. 전투 화면 — "시각 밀도"

### 7-1. 전투 카드 = **미니 타일**
- 컬렉션 풀카드(280×490) → 전장 미니 타일(80×100)
- 정보: **ATK/HP만**
- 상태 뱃지는 우하단에 **작은 아이콘 + 턴 숫자**

**RoF 현황**: `.td-slot` 이미 이 구조 ✅.

### 7-2. "턴 예고"
- 상대가 다음 턴 뭘 할지 **아이콘 예고**
- 공격: ⚔️ 숫자
- 방어: 🛡️ 숫자
- 버프: ✨ 아이콘
- 플레이어가 **반응 플레이** 가능 (단순 운 X)

**RoF 미구현** → 이미 sts1 스터디에서 제안됨. 모바일 특히 중요 (천천히 보며 판단).

### 7-3. 피해 피드백 체인
1. 공격 시작 → 카드 살짝 **좌측 이동** + **빨간 잔상**
2. 피격 카드 → **하얀 플래시 0.1초**
3. HP 숫자 **롤링 감소** (즉시 교체 ❌)
4. 플로팅 "-X" 빨강 숫자
5. HP 바 **1초 진행** 로 부드럽게 감소

**RoF 현황**: 일부 있음. **HP 롤링 애니메이션** 은 CardComponent 에 추가 가능.

## 8. 등급 연출 — "희귀도 쾌감"

### 8-1. 뽑기 연출 5단계
1. **카드 뒷면**: 다크 배경 + 축소
2. **빛 모임**: 카드 주변 파티클 수렴
3. **등급 선언**: 배경 색상 변화 (전설=보라, 신=주황)
4. **카드 전개**: 회전 + 확대
5. **정체 공개**: 일러스트 + 이름

- 일반 등급: 1~2단계만 (0.5초)
- 전설: 전체 5단계 (2초)
- 신: 전체 5단계 + 화면 풀스크린 점유 (3초)

**RoF 현황**: 등급 공개 연출 `RARITY_REVEAL_EFFECT.md` (구 GACHA_EFFECT.md, 2026-04-22 가챠 폐기 후 누적조각·랭크지급에 재사용) ✅.

### 8-2. 등급별 **사운드 톤**
- 일반: 나무 소리
- 희귀: 금속 소리
- 전설: 수정 + 합창
- 신: 종 + 오르간

**RoF 현황**: `30_sfx.js` 에 일부 구현.

## 9. 정보 밀도 관리 — "뺄 때 더 명료"

### 9-1. 화면당 주요 정보 5±2
- 한 화면에 **플레이어 눈이 머무는 영역 5~7개**
- 그 이상이면 눈이 길을 잃음
- "부가 정보" 는 **탭/호버** 로 숨김

### 9-2. 여백은 **의도된 빈 공간**
- 카드 주변 **10~20px 패딩** 필수
- 섹션 사이 **36~48px** 간격
- 밀도 높으면 고급스럽지 않음 (싸구려 느낌)

**RoF 적용 체크**: 현재 `.screen` 들이 여백 부족한 경향. 편집기로 재조정 중.

### 9-3. 색상 계층
- **1차 (중요)**: 앰버/금 — 행동 유도
- **2차 (정보)**: 텍스트 회색 스케일
- **3차 (강조)**: 위험=빨강, 긍정=초록, 신비=보라
- 동시 표시 색상 **최대 4종** 이하 유지

## 10. 즉시 적용 가능 체크리스트 (P0)

**이번 세션에 가능한 것**:

- [ ] 카드 hover 시 **앰버 글로우 + 이름 라벨** (특히 마을 빌딩)
- [ ] 중요 버튼 (⚔️ 도전자 찾기, ✅ 확정 등) **크기 차등** (높이 +30%, 배너형)
- [ ] HUD 정보 **3개 이하** 압축 (현재 5개 이상)
- [ ] 타이틀 배너에 **맥동 glow** 확장 (현재 타이틀만)
- [ ] 카드 앞면 능력 텍스트 **키워드 굵게** (✨ **도발**. 먼저 공격 받음)

## 11. 중기 검토 항목 (P1)

- [ ] 마을 배경 티어 1~3 발달 시각화
- [ ] 건물 레벨별 NPC + 인사말 (데이터 있음, 미연결)
- [ ] 적 다음 턴 예고 시스템 (전투 UX)
- [ ] HP 롤링 감소 애니메이션 (CardComponent)
- [ ] 전설/신 등급 뽑기 풀스크린 연출
- [ ] 호버 툴팁 3계층 (카드 → 요약 → 전체)

## 12. 금지 사항 (레퍼런스에서 배웠지만 복제 금지)

- ❌ 하스스톤 금속 + 보석 프레임
- ❌ 스냅의 특정 IP 캐릭터 의존
