# Phase 1: 핵심 루프 완성 계획

## 목표
현재 있는 기본 전투를 **재미있게 플레이 가능한 상태**로 다듬기.
새 기능 추가보다 **기존 시스템 안정화 + 밸런스 + 체감 개선** 우선.

## 현재 상태 (있는 것)
- [x] 로그인/회원가입/영웅선택
- [x] 기본 전투 (라운드 N = N번 공격, 오토배틀)
- [x] 로그라이크 구조 (전투 중 임시, 끝나면 리셋)
- [x] 라운드 사이 선택지 (유닛/스킬/유물)
- [x] 카드 시스템 (유닛24종, 스킬29종, 유물12종)
- [x] 출전권/유물력 시스템
- [x] 배치 (앞열/뒷열)
- [x] 적 AI (미러링 + 라운드 사이 선택)
- [x] 선술집 (유닛 구매)
- [x] 카드 강화 (골드)
- [x] 상태 확인 (카드/스킬/유물 보기)
- [x] 중세 BGM + 효과음
- [x] 다크 고딕 스타일 CSS (Cinzel 폰트 + 등급별 색상)
- [x] game-icons.net SVG 아이콘

## Phase 1 작업 목록 (우선순위 순)

### 1단계: 버그 수정 & 안정화
- [ ] **1-1. 전투 멈춤 버그 근본 해결**
  - try-catch로 임시 대응 중 → 원인 완전 제거
  - action='none'/'skill_only' 유닛의 전체 턴 플로우 재검증
  - bonusTrigger 스코프 문제 재확인
- [ ] **1-2. XP 시스템 검증**
  - 전투 후 카드별 XP 정상 적용 확인
  - 레벨업 시 스탯 반영 확인
  - 스냅샷 복원 시 XP 보존 확인
- [ ] **1-3. 출전 화면 빈 화면 버그**
  - 영웅만 있을 때 바로 배치로 가는 로직 검증
  - 유닛 2장 이상일 때 선택 UI 정상 작동 확인

### 2단계: 리그 점수 시스템
- [ ] **2-1. 리그 점수 도입**
  - Game 상태에 `leaguePoints` 추가 (저장/로드)
  - 승리 +10점, 패배 +5점
  - 상위 상대 가산점 (+5~20)
- [ ] **2-2. 리그 등급 표시**
  - 브론즈(0~99), 실버(100~299), 골드(300~599), 플래티넘(600~999), 다이아(1000~1499), 마스터(1500~1999), 신의영역(2000+)
  - 메인메뉴에 현재 리그 + 점수 표시
  - 리그 아이콘/색상
- [ ] **2-3. 적 매칭 리그 기반**
  - 같은 리그 내 ±1 티어 매칭
  - 상위 상대일 때 가산점 계산

### 3단계: 체력 포션 & 전투 개선
- [ ] **3-1. 4번째 선택지: 체력 포션**
  - 영웅 HP 50% 회복
  - 쿨다운 1라운드 (연속 사용 불가)
  - AI: 영웅 HP 30% 이하 시 자동 사용 (쿨다운 준수)
- [ ] **3-2. 전투 밸런스 조정**
  - 초반 (라운드1~3) 너무 운 좌우 → 기본 스탯 조정
  - 영웅 기본 HP 상향 검토
  - 평균 전투 길이 목표: 3~7라운드
- [ ] **3-3. 전투 UI 개선**
  - 라운드 진행 표시 더 명확하게
  - 영웅 HP 바 크게 강조
  - 적 영웅 이름/리그 등급 표시

### 4단계: 강화 이펙트 & 사운드
- [ ] **4-1. 카드 강화 연출**
  - 강화 성공 시 화려한 파티클 효과 (CSS animation)
  - 등급 상승 시 특별 이펙트 (금빛 폭발)
  - 강화 사운드 (Web Audio: 상승음)
- [ ] **4-2. 가챠 연출**
  - 카드 뽑기 시 뒤집기 애니메이션
  - 레어리티별 연출 차이 (전설=화려한 이펙트)
  - 뽑기 사운드
- [ ] **4-3. 전투 이펙트 강화**
  - 크리티컬 히트 → 화면 흔들림
  - 영웅 사망 → 슬로모션 + 어두워짐
  - 승리 → 금빛 배너 + 팡파레

### 5단계: 데이터 & 밸런스
- [x] **5-1. 초기 골드/경제 조정** ✅ (2026-04-11)
  - 초기 골드 2→5, 승리보상 10→12, 패배보상 3→5
  - 선술집 새로고침 2→1, 단련비용 level×3→level×2+1
  - 리그별 골드 보너스 7단계 세분화
- [ ] **5-2. 유닛 밸런스**
  - 각 유닛의 승률 데이터 (시뮬레이션)
  - 너무 강한/약한 유닛 조정
  - 종족별 균형
