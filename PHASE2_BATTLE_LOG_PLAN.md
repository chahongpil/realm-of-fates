## Part A: 전투 로그 (히스토리 바)

### 문제점
- 오토배틀이라 뭐가 일어났는지 놓치기 쉬움
- 특히 키워드 발동(급소, 흡혈, 부활 등)이 눈에 안 띔
- 승패 원인을 모르니 전략 개선이 어려움

### UI 디자인

#### 전투 중 로그 패널 (화면 우측 또는 하단)


## Part B: 전투 속도 조절

### 현재 상태
- 고정 속도로 전투 진행 (wait 함수 사용)
- 스킵 버튼은 있음 (skipReq)

### 속도 옵션

| 버튼 | 배속 | 대기시간 | 용도 |
|------|------|---------|------|
| ×1 | 기본 | 500ms | 처음 보는 전투, 연출 감상 |
| ×2 | 2배속 | 250ms | 일반 플레이 |
| ×3 | 3배속 | 150ms | 반복 전투, 빠른 진행 |
| ⏩ | 즉시 | 50ms | 결과만 보기 |

### UI 위치

```
┌─ 전투 화면 ──────────────────────┐
│ [HP바]           [×1][×2][×3][⏩] │  ← 우상단
│                                  │
│         전투 영역                 │
│                                  │
│                    [전투 로그 ▼]  │  ← 우하단
└──────────────────────────────────┘
```

### 구현

```javascript
// battleSpeed 변수 (기본 500ms)
Game.battleSpeed = 500;

// 속도 변경
Game.setSpeed(multiplier) {
  const speeds = {1: 500, 2: 250, 3: 150, 4: 50};
  this.battleSpeed = speeds[multiplier] || 500;
  // 버튼 활성 표시 업데이트
}

// 기존 wait(this.battleSpeed) 호출이 자동으로 속도 반영
```

## 웹 구현 시 고려사항

### 전투 로그
- `<div class="log-panel">` + `overflow-y:auto`로 스크롤 패널
- 각 로그 엔트리는 JS 템플릿 함수(`mkLogEntry`)로 재사용
- 키워드 발동 시 해당 엔트리에 CSS glow 클래스 추가
- 전투 요약은 별도 modal `<div class="modal-overlay">`

### 전투 속도
- `Game.battleSpeed` 변수 직접 조정 (현재 기본 200ms)
- 모든 `await wait(this.battleSpeed * speedMultiplier)` 자동 반영
- 속도 버튼: `data-speed="1|2|3|4"` 속성 + 클릭 토글
