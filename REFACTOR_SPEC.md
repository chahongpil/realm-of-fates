# Realm of Fates — 구조 리팩토링 스펙

> 작성일: 2026-04-10
> 브랜치: `refactor/structure-split` (작업 예정)
> 참고 문서: `ROADMAP.md`

## 목표

단일 파일 `index.html` (5285줄, HTML+CSS+JS 혼재)을 **협업 가능한 모듈 구조**로 분리한다. 게임 기능/이펙트/아트는 **변경하지 않는다**. 순수 구조 개편만 수행.

## 왜 하는가

- **협업 충돌 위험**: 현재 2명이 작업 중. 같은 파일 동시 수정 시 머지 충돌 상시 발생
- **탐색 비용**: 5285줄 단일 파일 검색/수정 시간 낭비
- **에디터 성능**: VSCode 렉, 신택스 하이라이팅 지연
- **향후 확장 기반 마련**: PixiJS 도입, 이펙트 고도화 전에 구조가 깔끔해야 함

## 핵심 제약 조건 (절대 지킬 것)

1. **빌드 도구 절대 금지**: Webpack, Vite, Rollup, esbuild, npm 등 일체 사용 금지
2. **순수 `<script>` 태그 분리만**: 여러 JS 파일을 `<script src="...">`로 순서대로 로드
3. **GitHub Pages 배포 구조 유지**: `index.html` 열면 그대로 작동해야 함 (`chahongpil.github.io/realm-of-fates/`)
4. **기능 0 변경**: 게임 플레이, 이펙트, 사운드, UI 어느 것도 기능이 바뀌면 안 됨
5. **에셋 경로 그대로**: `img/`, `snd/` 상대 경로 변경 금지
6. **의존성 추가 금지**: 현재 GSAP 외 새 라이브러리 추가 안 함

## 제안 파일 구조 (초안, 확정 아님)

```
realm-of-fates/
├── index.html              # HTML 뼈대만 (~400줄 목표)
├── css/
│   ├── base.css            # 공통 스타일, 버튼, 폰트
│   ├── screens.css         # 각 화면 (로그인/마을/전투 등)
│   ├── battle.css          # 전투 UI 전용
│   └── effects.css         # @keyframes, 애니메이션
├── js/
│   ├── data.js             # UNITS, SKILLS, RELICS 상수
│   ├── sfx.js              # 사운드 시스템
│   ├── ui.js               # 공통 UI, 모달
│   ├── auth.js             # 로그인/회원가입
│   ├── town.js             # 마을, 건물, 업그레이드
│   ├── tavern.js           # 술집, 영입
│   ├── battle.js           # 전투 로직
│   ├── cards.js            # 카드, 덱, 장비
│   ├── fx.js               # 파티클, Canvas 이펙트
│   └── game.js             # 메인 Game 객체, 진입점
├── img/                    # 그대로
└── snd/                    # 그대로
```

목표: **각 파일 500줄 이하**

## 현재 상태

- **브랜치**: 현재 `feature/defeat-animation` (패배 연출 작업 중, 미커밋)
- **리팩토링 브랜치**: `refactor/structure-split` (master에서 새로 파서 작업 예정)
- **팀원 상태**: 팀원에게 "로컬 작업 커밋 완료" 요청한 상태 — 완료 확인 후 리팩토링 시작

## 위험 요소

1. **스크립트 로딩 순서 의존성**: 예를 들어 `game.js`가 `data.js`의 상수를 참조하면 `data.js`가 먼저 로드돼야 함. 잘못하면 "undefined" 에러 발생
2. **전역 변수 공유**: 현재 모든 코드가 한 스코프에 있어서 전역 변수로 서로 참조. 파일 분리 후에도 전역에 유지해야 동작 (또는 네임스페이스 패턴)
3. **CSS 우선순위**: CSS 파일 로드 순서에 따라 스타일 덮어쓰기 달라질 수 있음
4. **GitHub Pages 캐싱**: 파일 경로 바뀌면 브라우저 캐시 이슈 가능
5. **기능 회귀**: 리팩토링 중 의도치 않게 뭔가 깨질 수 있음 → 회귀 테스트 체크리스트 필요

## 성공 기준

1. ✅ `index.html` 400줄 이하로 감소
2. ✅ CSS 파일 4개 분리 (base/screens/battle/effects), 각 500줄 이하
3. ✅ JS 파일 10개 분리, 각 500줄 이하
4. ✅ 게임 전체 기능 회귀 없음 (체크리스트 전부 PASS)
5. ✅ GitHub Pages에서 정상 동작 (배포 테스트)
6. ✅ 콘솔 에러 0개
7. ✅ 팀원 승인 (PR 리뷰 통과)

## 회귀 테스트 체크리스트 (리팩토링 후 모두 PASS 필수)

- [ ] 타이틀 화면 → 로그인 버튼 작동
- [ ] 회원가입 → 캐릭터 선택 → 마을 진입
- [ ] 기존 계정 로그인 → 마을 진입
- [ ] 마을 건물 클릭 → 각 화면(술집/교회/캐슬/대장간) 진입
- [ ] 술집에서 카드 영입
- [ ] 덱 보기 → 카드 상세 → 스탯 분배
- [ ] 전투 시작 → 전투 중 이펙트 정상 (슬래시, 마법, 크리티컬 등)
- [ ] 전투 승리 → 보상 화면
- [ ] 전투 패배 → 게임오버 화면
- [ ] 사운드 재생 (BGM + SFX)
- [ ] 저장/불러오기 (localStorage)
- [ ] 볼륨 조절
- [ ] 새로고침 후 데이터 유지

## 단계적 롤아웃 (권장)

한 번에 전부 하지 말고 **3단계로 쪼개서** 커밋/테스트:

### Phase 1: CSS만 분리
- `<style>` 블록 → `css/*.css` 4개 파일로 분리
- 리스크 낮음 (CSS는 기능 없음)
- 체크리스트 전체 테스트 후 커밋

### Phase 2: JS 데이터/상수 분리
- UNITS, SKILLS, RELICS 등 상수만 `js/data.js`로 이동
- 전역 변수로 유지
- 체크리스트 전체 테스트 후 커밋

### Phase 3: JS 로직 분리
- 기능별로 Auth, Game, Battle 등을 각 파일로 이동
- 로딩 순서 중요
- 가장 위험한 단계
- 체크리스트 전체 테스트 후 커밋

각 단계마다 **독립 커밋** → 문제 발생 시 해당 단계만 되돌리기 가능
