# 카드게임 UI 디자인 스킬 (학습 결과)

> 하스스톤, LoR, Fairtravel Battle 등 분석 기반
> 업데이트: 2026-04-12

---

## 핵심 원칙

### 1. 카드는 상태에 따라 변형됨
```
컬렉션/덱뷰: 풀사이즈 카드 (모든 정보 표시)
전장 위: 미니 타일 (ATK/HP만, 나머지 숨김)
손패(핸드): 중간 크기 (부채꼴 배치)
카드 탭/호버: 풀사이즈 팝업 (상세 확인)
```

### 2. 전장 레이아웃
```
적 영역: 화면 상단 40% (작게, 정보 최소화)
전장/보드: 중앙
아군 영역: 화면 하단 60% (크게, 플레이어 영역 강조)
```

### 3. 좌→우 흐름 (서양식 읽기 패턴)
```
왼쪽: 과거 (히스토리/묘지)
중앙: 현재 (보드, 플레이할 카드)
오른쪽: 미래 (마나, 턴 종료 버튼)
```

### 4. 가능한 행동 하이라이트
```
- 사용 가능한 카드: 밝게 빛남
- 마나 부족 카드: 어둡게
- 공격 가능 유닛: 테두리 빛남
- 타겟 선택 가능: 빨간 테두리
```

### 5. 모바일 최적화
```
- 최소 폰트 8pt 이상
- 줄간격 1.4~1.5배
- 장식 텍스트 < 능력 텍스트 (중요도 구분)
- 큰 버튼, 명확한 터치 타겟
```

## 전장 카드 vs 컬렉션 카드 차이

### 전장 카드 (미니 타일)
```
- 정사각형에 가까운 형태 (4:5)
- 일러스트 작게
- ATK (좌하단), HP (우하단)만 표시
- 이름 아래에 작게
- 프레임 최소화 (등급 색상 테두리만)
```

### 컬렉션 카드 (풀사이즈)
```
- 세로 긴 형태 (4:7)
- 일러스트 크게 (카드 면적의 60%)
- 모든 스탯 표시
- 능력 텍스트 표시
- 프레임 PNG 오버레이
```

## 참고 게임 UI 패턴

### 하스스톤
- 카드 프레임: 금속+보석 (우리는 다르게)
- 전장: 최대 7유닛, 가로 배치
- 손패: 하단 부채꼴
- 마나 수정: 우하단

### Legends of Runeterra
- 전장: 벤치 + 배틀필드 2줄
- 카드 최소화: 손패가 화면 가장자리에 살짝만
- 마나: 우측 원형 틱
- 모바일 퍼스트 디자인

### Fairtravel Battle
- 전장 유닛: 정사각 타일로 변환
- 최대 6유닛
- 좌→우 흐름 (과거→현재→미래)
- 플레이어 영역 60% / 적 40%
