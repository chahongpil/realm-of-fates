## 0. 전수 스캔 (관찰 + 한글 주석 원문)

> 모든 이미지에서 카드는 "번개 타이탄" (divine, `id:'titan'`) 동일 복제. 아래 모든 카드 내용은 동일:
> - 이름(상단 금색 밴드): **번개 타이탄**
> - 우상단 HP: **65**
> - 좌측 세로 3칸: **23 / 40 / 9** (ATK / DEF / SPD 순, 색상 박스)
> - 하단 설명 밴드: **뇌격: 전체8데미지**
> - 우하단 NRG: **16**
> - 프레임: 다크 레드/블랙 고딕 아치, 크랙 텍스처, 상단 중앙 금색 이름 밴드
> - 일러스트: 푸른 번개 후광 + 검은 타이탄 실루엣 (프레임리스)
> - 하단 가로 얇은 녹색 바 (HP 바로 추정)

### 1.png — 기본 2×5 그리드 (idle)
- **관찰**: 화면 전체에 동일 카드 10장이 **2행 × 5열** 배치. 상단 행(적), 하단 행(아군), 두 행 사이에 약간의 간격(약 카드 높이의 30%). 카드는 살짝 기울어진 부채꼴 없이 직립. 좌우 여백 균등. 배경은 거의 검정.
- **주석**: (없음)
- **해석**: 이것이 전투 기본 상태. 모든 카드가 "완전한 정보(이름·HP·3스탯·설명·NRG)"를 **상시 노출**. → `06-card-ui-principles.md` 의 "전장 미니 타일 80×100, ATK/HP만" 규칙은 **PHASE 3 에서 폐기**. 카드는 풀 정보를 가진 중간 크기 타일.

### 2.png — 1번(좌상단) 캐릭터 클릭 → 중앙 확대
- **관찰**: 좌상단 첫 카드가 사라지고, 화면 **정중앙에 동일 카드가 약 3배 크기**로 떠오름. 다른 9장은 그대로 제자리. 확대 카드는 주변에 은은한 금색 글로우.
- **주석 (원문)**:
  > "1번 번개타이탄 공격시 1번 카드가 중앙으로 이동하면서 약 3배 크기로 확대됨"
- **해석**: 캐릭터 클릭 → (1) 그리드 원본은 `visibility: hidden` 로 숨김, (2) 오버레이 중앙에 확대본 생성, (3) 스케일은 **약 3배** (현재 CSS `--bv2-card-focus-scale: 2.5` 와 불일치 → §8 플래그).

### 3.png — 확대 카드 아래 스킬/스펠 카드 5장 펼침
- **관찰**: 2.png 상태 + **확대 카드 하단을 물고** 스킬 카드 5장이 가로로 펼쳐짐. 스킬 카드는 기본 카드와 비슷한 톤(석재 프레임), 상단 중앙에 스킬 이름 금색 밴드, 좌측 세로 스탯 영역, 좌하단에 숫자(코스트), 일러스트는 인물형(회색 전사 흉상). 스킬 카드의 **상단 가장자리가 확대 캐릭터 카드의 하단 1/3 지점에 겹침**(Z 순서: 스킬이 앞).
- **주석 (원문)**:
  > "그리고 가지고 있는 모든 공격 및 스펠카드(여기서는 석궁병 카드가 공격 및 스펠카드라고 생각해줘) 보여줌. 기본 스펠 및 공격 카드들이 캐릭터 카드보다 앞에 보여지는걸로 하자"
- **해석**: 스킬 카드는 정확히 5슬롯 사전 선언. 캐릭터가 가진 스킬 수가 5 미만이면 나머지는 빈 슬롯(visibility hidden, 공간 유지). Z 순서 **스킬 > 캐릭터 확대본**.

### 4.png — 스킬 카드 위에 마우스 커서
- **관찰**: 3.png 와 동일 레이아웃, **중앙 스킬 카드 위에 흰색 마우스 커서 아이콘** 오버레이. 아직 호버 상태 변화는 없음(크기 불변).
- **주석 (원문)**:
  > "여기서 마우스를 만약 캐릭터 카드 위에 올리면"

  (참고: 주석 텍스트는 "캐릭터 카드"라 썼지만 커서 위치는 스킬 카드 위. 사용자 말실수로 보이며 다음 이미지 5.png 와 연결지으면 "스킬 카드 호버"가 의도로 보임)

### 5.png — 캐릭터(확대) 카드 위에 커서 → 더 확대
- **관찰**: 4.png 에서 커서가 **확대 캐릭터 카드 위에 얹힘**. 확대 카드가 **한 단계 더 커짐** (약 3.2~3.4배 느낌), 금색 글로우 강화. 그 외 레이아웃 동일.
- **주석 (원문)**:
  > "여기서 마우스를 만약 캐릭터 카드 위에 올리면 위로 더 확대되는 애니메이션. 위치는 내가 지금 정해진 대로"
- **해석**: 이미 중앙 확대된 캐릭터 카드도 호버 시 추가 확대(+살짝 상승). "위치는 정해진 대로"는 좌표 이동 없이 제자리 스케일만 증가.

### 6.png — 다른 데로 커서 이동 → 원래 크기로 축소
- **관찰**: 커서가 **그리드 첫 카드(빈 슬롯 자리)** 로 이동. 확대 카드가 5.png 보다 줄어들었으나 **2.png 기본 확대 상태보다 더 작음** — 즉 "아까 천(千)" 정도로 축소.
- **주석 (원문)**:
  > "다시 다른데로 마우스 돌리면 크기는 아까 천으로 줄어듬"
- **해석**: "아까 천" = "아까 초기값으로" 추정. 호버 해제 시 **2.png 의 3배 크기 상태로 복귀**, 그 이상 작아지지는 않음(그리드 원본 카드로 돌아가는 게 아님 — 여전히 중앙 확대 상태 유지). → 스케일 전이: base(1x) → focus(3x) → hover(3.3x) → focus(3x).

### 7.png — 커서가 스킬 카드(하단 첫 번째) 쪽으로 이동
- **관찰**: 커서가 **하단 스킬 카드 첫 번째 위치 근처**. 스킬 카드는 아직 반응 없음.
- **주석 (원문)**:
  > "만약 마우스를 1번 공격 및 스펠 카드에 갖다대면"

### 8.png — 1번 스킬 카드 호버 확대
- **관찰**: 7.png 의 1번 스킬 카드가 **크게 확대되어 상단으로 떠오름**. 확대된 스킬 카드의 내용이 선명히 보임: 이름 "석궁병", 우상단 HP "7", 좌측 스탯 "4 / 0 / 1", 설명 "관통: 방어무시". 확대된 스킬 카드의 **상단 가장자리가 확대 캐릭터 카드의 중간 지점**에 겹침. Z 순서: 스킬 > 캐릭터.
- **주석 (원문)**:
  > "이정도 크기로 확대되서 보여지는걸로 하자"
- **해석**: 스킬 카드 호버 시 ≈ **2배 스케일** + 수직으로 위쪽 이동(캐릭터 카드 쪽으로 밀고 올라감). 현재 CSS `--bv2-card-hover-scale: 1.15` 및 `.bcf-skill-card:hover { scale(1.15) }` → **불일치, 너무 작음**. §8 플래그.

### 9.png — 좀 더 명확한 스킬 카드 호버 (Z 순서 강조)
- **관찰**: 8.png 와 거의 동일하지만 **확대된 스킬 카드가 확대된 캐릭터 카드 위를 더 많이 덮음** (중앙 카드의 상단 1/3 지점까지 침범). 스킬 카드는 제일 앞 레이어.
- **주석 (원문)**:
  > "2번에 갖다대면 2번이 커지는데 1번이 다시 작아지고 제일 앞에 레이어 위치하는거로 하자. 살짝 위로 올라오면서 확대되어져 간다. 아래로는 팔릴(팔릴=펼쳐질?) 수가 있으니"
- **해석**:
  - 여러 스킬 카드를 순차 호버 시: **이전 호버 카드는 복귀 + 새 호버 카드가 확대 + 최상위 Z**.
  - 확대 방향: "**살짝 위로 올라오면서**" (아래쪽은 다른 스킬 카드가 펼쳐져 있으므로 충돌 방지).
  - [불명확] "팔릴" 오타 → "펼쳐질"로 추정.

### 10.png — 어떤 카드 클릭 → 모든 카드·적 페이드아웃 + 사용 카드 선택
- **관찰**: 화면 전체가 **어둡게 덮임**. 커서가 하단 중앙 근처. 확대 캐릭터 카드·스킬 row·그리드 모든 카드가 **회색 실루엣 톤으로 침잠**. (즉 사용자가 "선택 모드"에 진입했음을 시각적으로 표현)
- **주석 (원문)**:
  > "그래서 카드를 클릭하면 카드는 모든 카드(캐릭터 및 적)가 사라지는 애니메이션 나오고 사용 할 카드를 선택할 수 있음"
- **해석**: 스킬 선택 후 **타겟 선택 대기 상태**. "사라지는" 은 완전 제거가 아닌 **dim/desaturate** 로 해석. 현재 구현 없음 → §8 플래그.

### 11.png — 공격 카드일 경우: 적군이 회색, 본인 카드 포함
- **관찰**: 10.png 와 유사하지만 **아군 카드들이 회색으로 dim**. 적군(상단 행)은 **dim 해제, 공격 가능 후보로 활성**. 커서는 여전히 하단.
- **주석 (원문)**:
  > "공격카드일 경우(적만 선택가능할 경우에는 아군이 회색 처리됨, 본인 카드 포함)"
- **해석**: 타겟 선택 단계에서 **불가 대상은 전부 dim**. 공격 스킬이면 아군(+본인) dim, 적만 선명. 현재 구현은 적 카드에 `is-target-valid` 펄스만 추가할 뿐, **아군 dim 클래스 없음** → §8 플래그.

### 12.png — 지원 카드일 경우: 적군이 회색, 본인 포함
- **관찰**: 11.png 의 반대. **상단 행(적)이 회색 dim**, 하단 행(아군)과 확대 카드가 선명.
- **주석 (원문)**:
  > "지원카드일 경우(아군만 선택 가능할 경우에는 적군이 회색 처리됨, 본인 카드 포함) → 행동이 끝나면 화색처리되는거처럼 만약 행동 가능하군이 있으면 선택 가능하군이 다시 원래 색 갖는걸로 하자 (이때만)"
- **해석**:
  - 스킬의 `target` 필드에 따라 dim 대상 반전.
  - "본인 카드 포함": 자기 자신에게 사용 불가한 스킬이면 본인도 dim. 자가 버프면 본인도 활성.
  - 행동 종료 후 "이미 행동한 유닛"은 dim 유지(회색), 아직 행동 안 한 아군은 다시 선명 — 이는 **큐잉 모드(30초)에서의 유닛 순환 선택** 로직을 시사.

### 13.png — 타겟 카드에 마우스 호버 → 1.2배 확대 + HP 프리뷰
- **관찰**: **상단 적 첫 번째 카드**가 약 **1.2배 확대** + 위로 약간 떠오름. 해당 카드의 HP 숫자가 **65 → 57** 로 표시 (좌하단 근처에 "65 → 57" 형태). 커서가 해당 카드 위. 나머지 아군·확대 캐릭터는 dim 유지.
- **주석 (원문)**:
  > "공격 카드일 경우 마우스 갖다대면 적 카드가 호버되고 1.2배 정도 확대되면서 가장 맨 앞 와이어(?)로 나오고(아마도 와이어=레이어) 포인터는 항상 앞임.
  > 공격력이 2라드면 방어력 계산하여 상대방이 타격 받고 줄어든 최종 변경될 결과값도 미리 보여주는걸로 하자.
  > 지원 카드일경우도 마찬가지로 팁2 카드를 눌리고 마우스 갖다 대면 생명력이 65가 아닌 67로 보여지게"
- **해석**:
  - 타겟 호버 시: 1.2배 확대 + Z 최상위 + HP 프리뷰(데미지 계산 후 값). 현재 `--bv2-target-hover-scale: 1.08` → **너무 작음** → §8.
  - HP 프리뷰는 **카드 내부에 직접 표시** (좌하단 근처 "65 → 57") — 현재는 `#battle-target-preview .btp-hp-delta` 별도 박스 → 위치 차이. §8.
  - "포인터는 항상 앞임" = 마우스 커서는 모든 레이어보다 위(CSS 기본). 별도 대응 불필요.
  - 지원(힐) 카드도 동일 메커니즘 — 증가값 표시(65 → 67).

### 14.png — 타겟 클릭 → 모든 카드 사라짐, 중앙에서 발사
- **관찰**: 13.png 에서 타겟 클릭 후 **모든 카드가 사라짐**(그리드 전체 dim→투명). 확대 캐릭터 카드만 중앙에 남음. 커서는 하단 중앙. 번개 타이탄 아래쪽에 위로 향한 화살표 실루엣(이펙트 출발 방향 지시) 힌트.
- **주석 (원문)**:
  > "그래서 사용할 대상을 클릭하면 카드는 모든 카드가 사라지는 애니메이션 나오고 중앙 카드에서 애니메이션 출발해서 공격하는 식으로"
- **해석**: 실행 페이즈 진입 — (1) 그리드의 모든 비관여 카드는 페이드아웃, (2) 공격자(확대 카드)만 남음, (3) 공격자 위치에서 이펙트 발사.

### 15.png — 발사: 확대 카드에서 주황 화살표 상승
- **관찰**: 확대 캐릭터 카드 상단 우측에서 **주황색 번개/화살 이펙트**가 위쪽으로 뻗어나감. 다른 카드는 여전히 사라진 상태.
- **주석 (원문)**:
  > "그리고 선택한 스킬 및 공격 애니메이션 재생 [타겟에게] (당연히 액션 소리도 재생)"
- **해석**: 공격 스킬에 따라 다른 비주얼(번개, 화염 등). 사운드 동시 재생.

### 16.png — 적 그리드 복귀 + 타겟 카드 흔들림 (공격 적중)
- **관찰**: 상단 적 행이 **다시 나타남** (단 커서 타겟이었던 1번은 여전히 살짝 기울어지고 위로 이동 상태 — 흔들림 중간 프레임). 이펙트는 중앙에서 여전히 상승 중. 하단 아군 행은 아직 안 보임.
- **주석 (원문)**:
  > "스펠이나 공격 맞은 [타겟은] 카드가 흔들리는 액션"
- **해석**: 피격 반응 = **카드 흔들림 애니메이션** (좌우 shake + 기울임). 현재 CSS `@keyframes bv2CardShake` 있음, 연결 필요.

### 17.png — 적 그리드 유지, 공격 카드는 기본 카드 크기의 가로 회전
- **관찰**: 16.png 와 유사하지만 **이펙트가 사라졌고** 타겟 카드는 피격 자세(기울어짐) 유지. 또 한 개 다른 카드가 회색 틸트 상태 — [불명확] 아마 연속 피격 혹은 "사용한 스킬 카드도 같이 흔들리는 좌우 액션".
- **주석 (원문)**:
  > "스펠이나 공격 맞은 [타겟은] 카드가 흔들리는 좌우 액션"

### 18.png — 사망 처리 추정: HP 0 → 카드 "스스로 사라짐"
- **관찰**: 16~17.png 와 유사한 적 그리드 레이아웃. 확대 캐릭터 카드 HP 슬롯 근처가 강조됨. 한 쪽 카드가 사라지는 전환 중.
- **주석 (원문)**:
  > "만약 적 hp가 다 없어지면 카드가 녹아내린다던지 터진다던지 액션을 넣기로 하자 <이건 내가 넣어줄게 일단 스스로 사라지는걸로?>"
- **해석**:
  - 사망 연출은 **스프라이트 애니메이션(멜트·폭발)** — 사용자가 나중에 추가 예정. 1차 구현은 **단순 페이드아웃** 으로 시작.
  - HP ≤ 0 판정 → 카드 사라짐.

### 19.png — 확대 카드 축소 + 모든 카드 원래 자리 복귀
- **관찰**: 중앙 확대 카드가 **축소되며 원래 그리드 슬롯(좌상단 1번)** 으로 되돌아가는 중. 화살표가 위로 → 아래로 향함(복귀 방향). 다른 카드들은 이미 2×5 그리드 원위치.
- **주석 (원문)**:
  > "그리고 다시 원래대로 크기가 줄어들면서 원래 자리로 돌아오는거"
- **해석**: 1턴 실행 종료 → idle 복귀. 애니메이션: 확대카드가 translate + scale 로 원래 그리드 좌표로 이동 후 `visibility` 복원.

## 1. 레이아웃 규칙 (모든 이미지 공통)

- **배경**: 거의 검정 (`#0a0812` 계열), 배경 이미지는 opacity 0.3 이하로 은은하게
- **그리드 구조**: **2행 × 5열**, 총 10장 (적 5 + 아군 5). 상단 적, 하단 아군
- **행간**: 카드 높이의 약 30% (현재 CSS `gap: 48px` 적정)
- **카드간 수평 간격**: 카드 폭의 약 10% (`gap: 14px` 적정)
- **기본 카드 크기**: 화면 1280×720 기준 약 **110×160 ~ 130×180** (현재 CSS 130×180 적정). 10장 수평 배치 + 여백 가능해야 함
- **확대 카드 중앙 위치**: 화면 정중앙에서 상단쪽으로 translateY(-8%) 정도 끌어올림(스킬 row 공간 확보)

## 4. 상태별 요구사항

### S0. idle (1.png)
- 2×5 그리드 상시 렌더. 모든 카드 풀 정보.
- 오버레이 전부 hidden.

### S1. char-focus (2~9.png)
- 선택한 캐릭터 카드 그리드에서 숨김 (`visibility: hidden`)
- 중앙에 확대본 (3.0x). 스킬 row 5장 하단에 펼침 (Z: 스킬 > 캐릭터)
- 호버 가능:
  - **확대 캐릭터 카드 위 호버** → 3.3x로 확대
  - **스킬 카드 위 호버** → 2.0x + 위쪽으로 translateY, Z 최상위
  - 여러 스킬 카드 순차 호버 시 이전 카드는 base 복귀
- 빈 스킬 슬롯은 `visibility: hidden` (공간 유지, Grid gap 안정)

### S2. skill-active (10~12.png)
- 스킬 카드 클릭 시 진입
- **모든 카드 dim** (desaturate + 밝기 0.4) + 중앙 확대 카드도 dim
- 스킬 타입에 따라 **유효 타겟만 dim 해제**:
  - 공격: 적만 활성, 아군+본인 dim
  - 지원(힐/버프): 아군만 활성, 적 dim (본인 포함 여부는 스킬 설정)
  - 전체 대상: 아군+적 모두 활성
- **이미 행동 완료한 유닛** 은 계속 dim 유지 (큐잉 모드에서 중복 행동 차단)

### S3. target-hover (13.png)
- 활성 타겟 위 호버 시:
  - 해당 카드만 1.2x + translateY(-8px)
  - Z 최상위
  - **카드 내부에 HP 프리뷰 표시** (현재 HP → 계산 후 HP, 좌하단)
  - 공격이면 감소, 지원이면 증가
- 다른 카드는 dim 유지

### S4. fire (14~15.png)
- 타겟 클릭 시:
  1. **모든 비관여 카드 페이드아웃** (그리드 전체 opacity 0)
  2. **공격자 확대본만 남음** (중앙)
  3. 공격자 위치에서 이펙트 스폰 → 타겟 위치로 이동
  4. 사운드 재생

### S5. hit (16~17.png)
- 이펙트 도착 시:
  - 타겟 카드 위치로 **그리드 카드만 복원** (비타겟은 아직 숨김)
  - 타겟 카드 **좌우 흔들림** (`bv2CardShake`) + 살짝 기울임
  - 데미지 숫자 플로팅 (현재 `.bhr-dmg` 4rem)
  - 치명타 시 히트스톱 300ms + 주황 크리티컬 팝

### S6. death (18.png)
- 타겟 HP ≤ 0:
  - 1차 구현: **단순 페이드아웃** (8.png 주석: "일단 스스로 사라지는걸로")
  - 2차: 멜트/폭발 스프라이트 애니메이션 (사용자 작업 예정)

### S7. return (19.png)
- 확대 캐릭터 카드가 **translate + scale 애니메이션**으로 원래 그리드 좌표로 복귀
- 복귀 완료 후 `visibility` 복원, idle 로 복귀

## 5. 인터랙션 시퀀스 (정답 경로)

```
idle
 ├─ [카드 클릭] ──→ char-focus (확대 3x + 스킬 5장 펼침)
 │                   ├─ [빈 곳/ESC] → idle
 │                   ├─ [캐릭터 호버] → 3.3x, 해제 시 3.0x
 │                   ├─ [스킬 호버] → 2.0x (이전 호버 복귀)
 │                   └─ [스킬 클릭] ──→ skill-active
 │                                       ├─ [ESC] → char-focus
 │                                       ├─ [dim 해제된 타겟 호버] → target-hover (1.2x + HP 프리뷰)
 │                                       └─ [타겟 클릭] ──→ fire (0.3~0.5s)
 │                                                          └──→ hit (0.3~0.5s, 흔들림+데미지)
 │                                                               ├─ [HP > 0] → return → idle
 │                                                               └─ [HP ≤ 0] → death → return → idle
```

## 8. 🚨 텍스트 기획서 vs 스토리보드 충돌 플래그

1. **카드 정보량 충돌** (🚨 중요)
   - `.claude/rules/06-card-ui-principles.md`: "전장 미니 타일 80×100, ATK/HP만"
   - 스토리보드 1.png: **130×180 추정, 이름/HP/ATK/DEF/SPD/desc/NRG 전부**
   - → 스토리보드 정답. 규칙 파일 업데이트 필요: "PHASE 3 전투 카드는 전장에서도 풀 정보 표시, 크기 130×180".

2. **확대 스케일 수치** (🚨)
   - `06-card-ui-principles.md`: "중앙 무대 확대 ≈ 2.5배"
   - 스토리보드 2.png 주석: **"약 3배"**
   - CSS 변수: 2.5
   - → 사용자 확인 필요. 스토리보드 원문 "약 3배" 가 더 명확. CSS 3.0 으로 변경 권장.

3. **스킬 카드 호버 확대** (🚨)
   - 현재 CSS: 1.15배
   - 스토리보드 8·9.png: **≈ 2배 (사실상 풀카드 크기)**
   - → 현재 구현 불일치. 대폭 수정 필요.

4. **타겟 호버 스케일** (🚨)
   - 현재 CSS: 1.08
   - 스토리보드 13.png 주석: **"1.2배 정도"**
   - → 1.2 로 변경.

5. **HP 프리뷰 위치/방향** (🚨)
   - 현재: 화면 상단 별도 박스, 감소 방향만
   - 스토리보드 13.png: **타겟 카드 내부, 증가/감소 모두**
   - → 카드 내부 오버레이로 이전 필요.

6. **"dim 페이즈" 전체 부재** (🚨 가장 큰 공백)
   - 스토리보드 10·11·12.png 가 명시한 "모든 비관여 카드 회색 처리" 상태가 **현재 구현에 전혀 없음**
   - `is-target-valid` 펄스만 있음 → 반대 접근 필요(**기본 dim + 활성 해제**)

## 9. 미해결 질문 → 사용자 확정 (2026-04-14)

1. [9.png "팔릴"] → **"짤릴"** 의 오타. 의미: 아래로 확대하면 이미지가 화면 밖으로 짤려서 위로 확대/이동한다는 뜻. (원문 "아래로는 짤릴 수가 있으니")
2. [6.png "아까 천"] → "다시 다른 빈 곳으로 마우스를 두면 상대 카드가 살짝 줄어든다" = **non-hover (호버 해제)** 로 돌아간다는 의미. target-hover(1.2x) → 기본(1.0x) 복귀.
3. [2.png] 확대 배율 "약 3배"는 수사적 표현. **1.png 기본 카드와 2.png 확대 카드 실측 비교 필요** → 이번 세션에서 직접 측정한 결과 **≈ 2.0배** (그리드 카드 ~95px → 확대 카드 ~190px). **확정: `--bv2-card-focus-scale: 2.0`**
4. [13.png] HP 프리뷰 형식 → **최종값만 표시**. 예: `65` → `57` (계산은 유저가 암산). `.bv2c-hp-val` 텍스트를 직접 덮어쓰기, 호버 해제 시 원복.
5. [11.png] 공격 스킬 dim → **본인 + 아군 전체 dim, 적만 valid**. 이유: 공격 카드는 아군을 공격할 수 없으므로 대상 불가. 반사/카운터 예외 없음(현재 스코프 밖).
6. [14.png] "모든 카드가 사라지는 애니메이션" → **타겟 확정 후 모든 카드 사라짐. 중앙의 확대(캐스터) 카드만 남음**. 스킬 카드 5장 + 그리드 9장(타겟 포함) 전부 opacity 0. `.battle-stage-grid.is-fire-mode .bv2-card:not(.is-actor)` → `opacity:0`.
7. [18.png] 사망 애니메이션 → **1차는 페이드아웃만**, 향후 본격 작업으로 미룸. 확정.
