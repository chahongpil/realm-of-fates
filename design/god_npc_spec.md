## 1. 이관 구조

### 1.1 트리거

- 시즌 종료 시 **원소별 1 위 + 종합 1 위** 자동 산출 (7 명/시즌)
- 1 위 플레이어 = 신좌 계승자 → 본인 캐릭터 = 다음 시즌 해당 신좌의 **점유자 NPC** 로 이관

### 1.2 이관 데이터

- 플레이어 캐릭터 외형 (프리셋 + 원소 이펙트)
- 최종 카드·유물 덱 (그 시즌 마지막 구성)
- 최종 스탯 (레벨·HP·ATK·등)
- 본인이 작성한 **"유언" 대사** (§ 6 참조)
- 등극 당시 시즌 번호·세대 번호

### 1.3 원 플레이어 처우

- 본인 계정 유지 → 새 캐릭터 생성 재시작
- 이전 캐릭터는 본인 컬렉션에서 "기증됨" 상태 아이콘
- 프로필에 "제N세대 [신좌] 계승자" 칭호 영구 부착 (누적 가능)

---

## 2. 활성 범위 — 최근 5 세대

### 2.1 원칙

- **각 원소 신좌 당 최근 5 세대만 활성** = 서버 실시간 등장
- 불 신좌 예: 현 세대 그라힘 + 최근 4 세대 = 5 명 동시 활성
- 5 세대 × 6 원소 신좌 + 5 세대 × 7 번째 신좌 = **35 활성 NPC**
- 6 번째 세대 계승 시 → 가장 오래된 세대가 **명예의 전당으로 이동**

### 2.2 15 개월 근거

- 1 시즌 = 3 달 확정 → 5 × 3 = 15 개월
- Marvel Snap 유사 신전 UI 슬롯 수용 한계 ~5 (UI 과부하 방지)
- 장기 플레이어 애착 vs 신진대사 균형

---

## 3. 이전 세대 처우 — 명예의 전당

### 3.1 UI 위치

- 성 허브 내 **"명예의 전당"** 공간 (2번 작업 "성 허브 퀘스트 설계" 에서 상세)
- 각 원소 회랑 + 7 번째 신좌 중앙 홀
- 세대 순서로 정렬, 검색·필터 가능

### 3.2 전시 정보

- NPC 명 (`제N세대 [신좌] — [캐릭터명]`)
- 외형 시각화 (카드 이미지)
- 당시 스탯·덱 스냅샷
- 유언 대사 (검수 통과된 시그니처 문장)
- 등극 기록 (시즌 번호, 당시 랭크 포인트, 주요 업적)

### 3.3 상호작용 제한

- **만날 수 없음** (싱글 런 조우 불가, 초대장 구매 불가)
- 보기만 가능 — "역사 기록" 으로서의 영원성

---

## 4. 상호작용 모델 — 조합

### 4.1 확률 조우 (기본)

- **싱글 런 중 랜덤 이벤트** (StS 이벤트 조우식)
- 발동 확률: **액트당 10~15%** (튜닝 대상)
- 발동 시 "이전 세대 네크리온이 당신을 지켜보고 있다" 류 이벤트 등장
- 선택지:
  - **대화** — lore 확장, 보상 소량
  - **도전** — 전투, 보상 대량, 위험도 ↑
  - **무시** — 그냥 지나감

### 4.2 초대장 구매 (의도적 확정 조우)

- 성 허브 **"옛 신의 초대장"** 아이템 판매
  - 화폐: 골드 + 신의 잔재 소량
  - 지정 선택 가능 (제N세대 X 신좌)
  - 싱글 런 중 반드시 조우 확정
- 용도: 특정 이전 세대와의 도전·대화·업적 달성

### 4.3 보상

- **대화**: 업적 포인트 + lore 해금
- **도전 승리**: 신의 잔재 + 전설 유물 조각 + 칭호 ("제N세대 X 를 격파한 자")
- **도전 패배**: 실타래 손실 없음 (주 랭크 게임과 분리)

## 5. 스탯·카드 스냅샷

### 5.1 고정 시점

- 이관 순간 = 시즌 종료 시점의 최종 구성
- 이후 원 플레이어가 새 캐릭터로 성장해도 NPC 는 업데이트 X
- "그 시즌의 순간 포착" 개념

### 5.2 밸런스 재조정

- NPC 스탯은 게임 밸런스 변동에 맞춰 **운영 측에서 재조정 가능** (AI 난이도 튜닝)
- 원본 스냅샷은 갤러리 전시용 보존, 전투용 스탯은 별도 테이블
- "그 시즌엔 이 스탯이었음" 기록은 영구

## 6. 대사·플레이버 — C+ 하이브리드

### 6.1 구조

- 기본 대사 = **템플릿 선택 3~5 개**
- 시그니처 = **자유 입력 1 문장 (60 자)**
- 둘 다 NPC 대사 풀에 섞여 랜덤 출력

### 6.2 템플릿 50~60 문장

- 신별 8~10 개씩 (lore-bible v2 § 12.2 톤 적용)
- 카테고리:
  - 조우 인사 (2~3)
  - 도전 수락 (2~3)
  - 승리 시 (1~2)
  - 패배 시 (1~2)
- 예시 (네크리온 계열): _"계약은 공정했다"_ / _"너는 대가를 치렀다"_ / _"내 서명이 기다린다"_

### 6.3 시그니처 문장 (자유 입력)

- **최대 60 자**
- 등극 UI 에서 플레이어 직접 입력
- 템플릿 대사와 섞여 랜덤 출력
- 플레이어가 입력 거부 시 → 템플릿만으로 NPC 완성 (강제 X)

### 6.4 필터·검수 큐

- **1 차**: 자동 필터 (욕설·URL·명시적 홍보)
- **2 차**: 운영자 수동 검수
  - 시즌 종료 후 **다음 시즌 시작 전 3 주 공백** 에 일괄 검수
  - 시즌당 7 명 × 연 4 시즌 = **연 28 문장**
  - 운영자 1 명이 시즌 전환일 10 분 처리
- **검수 가이드**:
  - 거부: 욕설, 정치·종교·개인 비난, 외부 홍보
  - 통과: 그 외 전부 (개성·유머·철학 표현 자유)
- **거부 시**: 본인 알림 + 재입력 기회 1 회 → 그래도 거부 시 템플릿만 사용

### 6.5 검수 지연 = 게임 리듬과 일치

- 시즌 시작 직후 이전 세대 신 NPC **즉시 출현 X** (3 주 검수 기간)
- 대신 "옥체 재봉 컷신 + 새 유언 배분" 시즌 오프닝 연출 몰입
- 검수 완료 = 약 시즌 1/4 시점에 이전 세대 활성화 → "다음 주 부활" 기대감으로 활용

## 7. NPC 명명 규칙

### 7.1 형식

`[세대 번호] [신좌 이름] — [플레이어 캐릭터 이름]`

**예**:
- **제3세대 네크리온 — 카이엔**
- **제7세대 브론테스 — 유스티나**
- **제1세대 일곱 번째 자리 — 아론**

### 7.2 세대 계산

- 제1세대 = lore-bible v2 기본 신 (네크리온 타락천사 등)
- 제2세대 = **게임 출시 첫 시즌 계승자**부터 시작 (서비스 런칭 시점)
- 계정별 "내가 몇 세대 계승자였는가" 자동 추적

### 7.3 신좌 이름 각색 (선택)

- lore-bible v2 § 0.3 규칙 적용
- 새 점유자가 원하면 "암흑 신좌의 새 이름" 지정 가능 (네크리온 → 네오크리온 등)
- 기본값 = 원래 신좌 이름 유지

## 8. 스폰 위치

### 8.1 정적 스폰 (갤러리)

- 각 원소 신전 내 **"역대 점유자 회랑"**
- 최근 5 세대 나란히 전시 (활성 + 만날 수 있음)
- 6 세대 이전은 "명예의 전당" 별도 공간 (전시 전용)

### 8.2 동적 스폰 (조우 이벤트)

- 싱글 런 액트 1~2 랜덤 이벤트 지점
- 초대장 사용 시 확정 조우 (액트 1~2 지정 가능)

## 9. 서버·DB 고려

### 9.1 저장 범위

- 활성 35 NPC × 전체 데이터 (외형·스탯·덱·대사 풀)
- 이전 세대 N → 전시용 경량 데이터만 (스탯·덱은 요약)

### 9.2 확장성

- 연 28 명 등극 × 서비스 10 년 = **280 NPC 누적** (전시 포함)
- DB 크기 부담 미미 (1 NPC 당 KB 수준)

### 9.3 스냅샷 압축

- 활성 → 이전 세대 이동 시 전투용 스탯 → 전시용 요약만 보존
- 덱·상세 카드는 요약으로 축소

## 10. 엣지 케이스

### 10.1 동점 시즌 1 위

- 시즌 종료 시 동점 → 타이브레이커 순서: 실 가닥 수 → 최대 치명타 → 싱글 런 완주 횟수
- 그래도 동점이면 **두 명 모두 등극** (그 시즌 한정, NPC 슬롯 임시 확장)

### 10.2 1 위 없는 시즌

- 최소 참가 기준 미달 → 유효 1 위 없음 → 현 점유자 유지 + 제N+1 세대 skip
- 다음 시즌에서 재개

### 10.3 플레이어 계정 삭제

- 원 플레이어 계정 삭제 시 NPC 는 **그대로 유지** (세계관 자산화)
- 단, 캐릭터명 대신 "이름 없는 계승자 #N" 로 표시

### 10.4 같은 플레이어 다중 등극

- 같은 원소·다른 원소 연속 등극 제한 없음
- 같은 계정 내 "제3·5·7세대 네크리온 계승자" 처럼 누적 칭호

## 11. 영향받는 시스템

### 11.1 UI

- 성 허브 — "역대 점유자 회랑" + "명예의 전당" (2번 작업에서 상세)
- 등극 시 — 유언 입력 UI (템플릿 선택 + 자유 입력)
- 싱글 런 — 조우 이벤트 UI (대화·도전·무시)
- 성 허브 상점 — 옛 신의 초대장 판매

### 11.2 DB

- `god_npc` 테이블 (활성 + 전시)
- `god_npc_dialogue` 테이블 (템플릿 + 시그니처)
- `god_npc_encounter_log` 테이블 (플레이어별 조우 기록)

### 11.3 운영 파이프라인

- 시즌 전환일 운영자 검수 큐 (10 분/시즌)
- 신고·이의 제기 시스템 (기존 대응 인프라 재활용)
