# 레퍼런스 게임 세계관 학습 노트 — 2026-04-21

> **세션 목적**: 대표님이 자율 30분 부여. "운명의 왕좌 — 일곱 번째 자리" IP 강화를 위해 레퍼런스 6~7개 게임의 세계관·서사 구현 방식을 수집·증류하고, 우리 IP에 맞는 적용안을 도출.
> **작성자**: 메인 세션 (docs-lore 작업)
> **범위**: 읽기 전용 리서치 + 본 문서 신규 작성만. 기존 기획서는 건드리지 않음.
> **후속**: 이 문서의 "적용안" 섹션은 대표님 검토 후 `lore-bible.md` / PHASE 기획서 / 카드 flavor text 로 분산 반영.

## 0. 우리 IP 현상태 (출발선)

- **세계관명**: "운명의 왕좌" / 부제 "일곱 번째 자리" (The Seventh Throne)
- **핵심 고유 장치**:
  - 창세 운명의 여신(Weaver) 이 **일곱 자리**를 남기고 잠듦
  - 여섯 자리는 원초의 여섯 신(그라힘/모라스/에이드라/비리얀/세라피엘/네크리온)이 차지
  - 일곱 번째는 비워둠 — 조건은 "필멸자 출신만"
  - 여섯은 일곱 번째 자리를 **두려워하며**, 자신의 **파편(= 카드)** 을 흘려 필멸자를 시험·견제
  - 플레이어 = 평범한 필멸자. 파편 수집 + 자신의 영웅 카드 신 등급 각성 = 일곱 번째 자리 등극
- **강점**: "파편 = 카드" 가 기계적 구조(rarity 5단계) 와 서사(상승의 단계) 를 이미 일치시킴
- **공백**:
  - 런 사이 서사 전달 장치 미정 (허브 NPC 대사 시스템 없음)
  - 카드 flavor text 톤·분량 기준 미정
  - 이벤트·선택지 내러티브 구조 미정
  - 여신의 존재감을 엔딩까지 0 으로 둔 채 몰입 유지 방법 미정
  - 신들 간 "관계 매트릭스" 는 있으나 **플레이 중 드러나는 방식** 미정

아래 레퍼런스 학습은 이 공백을 메우기 위한 카드 찾기.

## 1. Hades I & II (Supergiant) — 로그라이크 드립피드 서사

### 핵심 패턴
- **죽음 = 진행**. 실패가 서사 진행 레버. 매 런마다 허브(집) 대화 업데이트.
- **NPC 리액션**: 플레이어 최근 행동을 NPC 가 기억하고 멘션. ("그 무기 들고 갔던 네가 전기 여신한테 또 당했다고?")
- **10시간 분량 chained 대화**. 하나의 이벤트(에우리디케 만남) 가 다른 NPC(오르페우스) 대화 트리를 잠금 해제.
- **신별 시각·말투 차별화**: 각 신마다 **단일 색**이 부여 (붉은 아레스, 푸른 포세이돈 등). 말투도 고유 — 헤스티아는 불 말장난, 아르테미스는 자격지심 섞인 소외감.
- **보상 ≠ 선택**: 받은 부운(boon) 이 계보·관계를 드러냄. 아레스 부운 받으면 포세이돈이 "그 녀석 부운이야?" 하고 논평.

### 우리 IP 적용
- **성(Castle) / 선술집(Tavern) 을 "집" 으로 활용**. 런 종료 시 해당 화면에서 방금 런 이벤트에 대한 NPC 멘션.
  - 예: "그라힘 던전에서 세 번째 실패했다지? 그 드래곤은 너 정도로는 아직…" (선술집 주모)
  - 예: "모라스의 파편을 받았군. 잊은 게 있나?" (성의 현자)
- **런별 대화 드립피드**: 한 신의 파편을 처음 얻은 런 / 세 번째 런 / 열 번째 런에 따라 NPC 코멘트 다르게.
- **신의 색은 이미 확정** (원소 팔레트). 말투 가이드는 `lore-bible.md` 에 일부 있음 → **대사 샘플 확장 필요** (신당 30~50줄 목표, 현재 2~3줄).
- **부운 형식을 "파편 수집 시 신 본인 대사"** 로 차용: divine 카드 드롭 시 해당 신의 1줄 음성/텍스트 출력. 전설 카드는 그 신을 섬긴 영웅의 대사.

### 데이터 함의
- `js/` 에 `DIALOG_DB` 를 추가하되 트리거 키(run_count, god_id, event_id)로 인덱싱.
- 트랙 1(메인) 이 데이터 구조를, 트랙 5(docs-lore) 가 대사 본문을 작성하는 분업이 자연스럽다.

## 2. Slay the Spire II (MegaCrit) — 이벤트 내러티브 설계

### 핵심 패턴
- **66 이벤트** / 액트당 8~12개. 각 이벤트는 2~3 선택지.
- **장문 이벤트 존재**: 11-page 재판 이벤트 — "후회 vs 죄책감" 같은 추상 개념 선택.
- **디자인 이벤트**: "워크샵" 에서 플레이어가 직접 카드 하나를 설계(스탯 할당) 후 덱에 추가 → 내러티브와 기계를 융합.
- **보상 비대칭**: 한 선택은 **즉각적 기계 보상**(카드·골드), 다른 선택은 **장기적/감정적 보상**(영구 버프·저주 해제·NPC 플래그).
- **하이키 공유 이벤트**: 여러 액트 공유 이벤트로 지속성 확보 — 플레이어가 "이 이벤트 또 나왔네" 하는 순간 서사 반복 각인.

### 우리 IP 적용
- **던전 이벤트 타일** 도입 (Phase 2/3 던전 진행 중 1~3회 랜덤): 전투 대신 선택지 화면.
  - 템플릿 A: "파편 vs 정화" — 파편 받기(카드 1장) vs 다음 전투 디버프 해제
  - 템플릿 B: "맹세 vs 계약" (세라피엘·네크리온 대립 반영) — 한쪽 신 가호 +, 반대편 -
  - 템플릿 C: "기억 포기" (모라스 시험) — 특정 카드 한 장 영구 삭제 → 보상 HP 영구 +
- **디자인 이벤트**: "대장간" 을 이미 가지고 있으므로 여기에 "카드 설계" 이벤트 변형 추가 가능 — 무기/방어구/도구 중 하나 슬롯 스탯 커스터마이즈. (트랙 2 밸런스 개입 필요)
- **시험의 결과 = 영구 플래그**: 그라힘의 "살아남아라" 시험을 특정 확률로 통과하면 플레이어 프로필에 "그라힘의 인정" 플래그 → 이후 모든 불 카드 +1 스탯. 신별 6개 인정 플래그가 엔딩 조건 중 일부.

### 집필 원칙 (StS2 기준)
- 이벤트 본문은 3~6 문장 이내. 장문은 특별 이벤트에만.
- 선택지 이름은 **명사형 2~3 단어** ("후회" / "죄책감" / "침묵"). 풀어쓰지 않음.
- 선택 후 결과 문장은 1~2 줄로 짧게. 보상은 즉시 팝업.

## 3. Genshin Impact (HoYoverse) — 7 원소국가 구조

### 핵심 패턴
- **7 나라 = 7 원소 + 7 이념**. 원소만 배치하면 단조 → 각 나라에 **이념 라벨** 을 붙임 (몬드=자유, 리월=계약, 이나즈마=영원, 수메르=지혜, 폰타인=정의, 나타=전쟁, 스네즈나야=사랑).
- **현실 문화 매핑**: 몬드=독일, 리월=중국, 이나즈마=일본, 수메르=남아시아/중동, 폰타인=벨 에포크 프랑스 — 문화 인상을 원소 위에 덮어 쓰레없이 차별화.
- **신의 권력 3 원천**: (a) 영토 지배, (b) 백성의 신앙, (c) 천리(天理)가 원주인(Seven Sovereigns)에게서 빼앗아 부여한 원소 권능. → **신은 약해질 수 있다** 는 서사 레버.
- **아르콘의 한계**: 천리와 계약 관계. 완전히 자유롭지 않음 — 긴장의 원천.

### 우리 IP 적용
- **여섯 신 각각에 "이념" 라벨을 한 단어로 붙일 것** (lore-bible.md 의 "시험" 을 이념화):
  | 신 | 이념 한 단어 | 현실 문화 참고 |
  |---|---|---|
  | 그라힘 | **정직** (잔인하나 솔직) | 바이킹·몽골 기마 |
  | 모라스 | **망각** | 그리스 지하 신전 / 일본 바다마을 |
  | 에이드라 | **인내** | 티베트·안데스 고원 |
  | 비리얀 | **속도** | 로마 호위군·현대 대도시 |
  | 세라피엘 | **맹세** | 비잔틴·중세 수도원 |
  | 네크리온 | **계약** | 19세기 런던·뉴욕 월가 |
- **문화 인상**은 **과하지 않게** — 우리 하네스 규칙(금지: 오크/엘프/전형적 서양 중세) 을 지키려면 현실 매핑은 카드 이름·배경 아트의 **실루엣 수준**에서만. 구체적 건축·의상은 피함.
- **신의 한계 서사 레버**: 여신이 잠들면서 **신들의 힘도 상한** 이 생김 — 파편이 세상에 흘러나오는 이유 = 신들도 자신을 완전히 유지할 수 없음. 이 설정은 플레이어가 신을 **극복 가능한 존재** 로 인식하게 해주고, 일곱 번째 자리 등극의 개연성을 지지.

### 피할 것
- 현실 문화 매핑을 "그 나라 사람이 뻔히 보이게" 하면 하스스톤·원신 표절 인상. 우리는 **신의 성격 + 원소 감정선** 우선, 문화 레퍼런스는 아트 방향의 백그라운드 자료로만.

## 4. Honkai: Star Rail — 아이온(Aeon) = 철학의 화신

### 핵심 패턴
- **아이온 = 철학**. Destruction(파멸) / Preservation(존속) / Erudition(지혜) / Harmony(화합) 등. 신=감정이 아니라 신=개념.
- **아이온 본체는 부재·간접**. 그들의 **길(Path)** 을 추종자들이 걸음. 플레이어 캐릭터의 Path 가 곧 전투 포지션 (Preservation=탱커, Harmony=버퍼, Destruction=딜러).
- **기계와 서사 1:1 매핑**: 서사적 이념이 곧 전투 스타일. "보존의 길을 따르는 자는 방패를 쓴다" 가 **lore 이면서 game mechanic**.
- **부재 중에도 존재감**: 아이온 본인은 거의 등장 안 하지만 전 세계 캐릭터가 그들을 언급 → 신화적 무게감 유지.

### 우리 IP 적용
- **여섯 신의 "시험" 을 역할 플레이스타일로도 매핑**:
  | 신 | 시험 | 제안 플레이스타일 |
  |---|---|---|
  | 그라힘 | 폭력 | 고 atk / 저 def / 광폭화 |
  | 모라스 | 망각 | HP 재생 / 손실 기반 스케일링 |
  | 에이드라 | 인내 | 고 def / 턴 지연 카드 |
  | 비리얀 | 속도 | 고 spd / 선제공격 |
  | 세라피엘 | 약속 | 고 nrg / 지원·버프 |
  | 네크리온 | 계약 | 고 luck / 흡혈·치명타 |
  - 이건 이미 `04-balance.md` 의 원소 보너스와 거의 일치. → **"이념 = 밸런스" 를 공식 문서화**하면 신화와 기계가 한 덩어리가 된다.
- **신 본인의 등장 최소화** — 아이온처럼 **신 카드(divine)** 는 "본체가 직접 내려온 1회성 간접 출연" 으로 프레이밍. 평소엔 NPC 가 그들을 "언급만". 이렇게 하면 신의 신비감 유지 + divine 카드의 출현 자체가 서사 이벤트.

### 놓치지 말 것
- 추종자가 신의 철학을 **왜곡**하는 서브 서사가 HSR·원신의 진짜 재미. 우리도 "세라피엘의 이름으로 과도한 숙청을 행하는 광신도" 같은 카드 설계가 가능 — **같은 신을 섬겨도 파편의 해석은 다를 수 있다** 는 축.

## 5. Legends of Runeterra / Magic: The Gathering — 카드 플레이버 텍스트

### 핵심 패턴
- **2줄 이내 / 1~2문장**. 능력 텍스트와 분명히 시각적으로 분리(이탤릭체).
- **인용 + 화자 명시 포맷**이 가장 많이 씀:
  > "그는 내 이름을 불렀다. 그것이 첫 번째 실수였다."
  > — 네크리온, 밤의 계약자
- **관계 드러내기**: 다른 캐릭터와의 관계·라이벌·연인 언급이 세계관을 확장. ("치명적인 적을 두 명 이상 이름짓지 말라.")
- **Adventures with Chip** 같은 **시리즈형 플레이버**: 여러 카드의 플레이버가 하나의 미니 스토리 → 컬렉터 재미.
- **절대 금지 (MtG·LoR 기준)**: 게임 자체를 언급하지 않음("이 카드는 강합니다" 같은 메타 발언 금지). **4th wall 깨기는 Hearthstone 스타일 — 우리는 피함** (우리 디자인 규칙 `05-design-direction.md` 와 일치).

### 우리 IP 적용 — 카드 플레이버 텍스트 규칙 (제안)
1. **길이**: 1~2 문장, 최대 2줄. 40자 x 2줄 = 80자 이내.
2. **포맷 3종 중 하나 택**:
   - **인용 + 화자**: "…" — 화자, 호칭
   - **단문 관찰**: "숨이 얼어 공중에 박힌다."
   - **비유 선언**: "그는 약속이다. 그리고 약속은 칼이다."
3. **금지**:
   - 게임 메타 언급 ("이 카드는…", "당신의 덱에…")
   - 현대 유행어·팝 문화 (하스스톤식 유머)
   - 다른 프랜차이즈 크로스 레퍼런스
4. **권장**:
   - 신의 대사 샘플 (이미 lore-bible 에 2~3줄씩 존재) 을 divine·legendary 카드에 직접 인용
   - bronze·silver 카드 플레이버는 **필멸자 시선** — 신을 멀리서 관찰한 농부·병사의 한 마디
   - gold 카드는 **반신·성좌** — 신의 언어를 흉내 내는 영웅의 말
   - 같은 원소의 카드 5~10장을 묶어 **연결된 플레이버 스토리**(LoR 스타일 "Adventures with") 설계
5. **집필 분담**: 트랙 5(docs-lore) 가 원소당 10~20 플레이버 초안 → 트랙 1(메인) 이 데이터 주입.

### 샘플 (6원소 divine 카드 플레이버 초안)

## 6. Elden Ring / Hollow Knight — "부재하는 창조자" 아키타입

### 핵심 패턴 — Elden Ring
- **Greater Will = 먼 절대자**. 본인 등장 0. 사자인 Two Fingers 가 유일한 접점 (그것도 제한적).
- **Marika = 현현 + 봉인**. 한때 신이었으나 엘든 링을 깨고 에르드트리(세계 중심) 에 봉인됨. 신이지만 **부재** 상태.
- **Tarnished = 신에게 버림받았다가 소환된 자**. 플레이어 = 이류 / 변방 / 귀환자. 엔딩에 왕관을 쓸 수도, 거부할 수도.
- **Shattering = 세계의 중심 룬이 산산조각 → 파편(Great Runes) 이 반신들에게**. 세계의 구조가 파편 수집 구조와 동일.

### 핵심 패턴 — Hollow Knight
- **창조주의 오류**: Pale King 이 해결책(Vessel) 을 만들었으나 **자신이 만든 해결책에 감정을 가져 실패**. 신의 실수가 게임의 상황.
- **순수한 그릇**: Vessel 은 **감정이 없어야** 봉인이 된다는 역설. 아버지의 사랑이 순수를 훼손.
- **희생된 세대**: 수십의 Vessel 형제자매들이 버려져 죽음. 플레이어 = 살아남은 버려진 자.

### 우리 IP 적용
1. **운명의 여신을 "Greater Will 모델"로 유지**: 대사·등장 0. 사자도 필요 최소로. 우리 lore-bible 이 이미 이 방향 — 계속 유지. **엔딩 전까지 딱 한 번 등장 시점만 설계** (예: 모든 신을 이긴 뒤 신 등급 각성 순간).
2. **여신의 "흔적" 을 배경에 깔기**: 엘든 링의 Two Fingers 처럼, 던전·허브 어디에도 **실**·**베틀**·**직조된 무늬** 모티프가 등장. 보이지 않는 여신의 손이 닿았음을 시각 언어로만 암시. (아트 트랙 1과 공유할 시각 규칙)
3. **"일곱 번째 자리" = Great Runes 구조**: 여섯 신의 파편이 모두 수집되어야 일곱 번째 자리가 완성. → 엔딩 조건을 "6 신 모두에게서 인정 받기" + "자신의 영웅 카드 신 등급 각성" 두 축으로 명시.
4. **Hollow Knight 교훈 — 플레이어의 "결함"을 엔딩 변수로**: 플레이어가 특정 신에게 너무 의존(파편 편중)하거나 모라스 시험으로 너무 많은 기억을 포기했다면 엔딩 분기 변화. "순수한 후계자" vs "불완전한 후계자" 등.
5. **"여신도 답을 모른다" 설정**: 엘든 링의 Greater Will 조차 완벽하지 않다는 암시(슬픔의 여신 Metyr 의 제한된 접촉) → 우리도 "여신이 잠들기 전에 이 결말을 원했는지 불분명" 을 남김. 플레이어가 여신의 의도를 **해석**할 여지를 남기는 것이 모호한 깊이를 만든다.

## 7. Inscryption — 카드 자체가 서사 매체

### 핵심 패턴
- **카드 뒷면에 메모·스티커·변색**: 카드 개체가 이야기의 물리적 증거.
- **메타 프레이밍**: "이 게임을 플레이하는 유튜버 Luke" 가 바깥 층. 플레이어 = Luke = 플레이어 3중 레이어.
- **해결할 수 없는 룰이 나타나면 이야기 분기**. 카드 룰 자체가 서사 트리거.

### 우리 IP 적용 — 부분 채용
- **Inscryption 급 메타 프레이밍은 우리 톤과 안 맞음**. 우리는 로그라이크 오토배틀러라 전통적 3인칭 세계 고수.
- **그러나 "카드 개체성"은 훔칠 만함**:
  - 특정 카드가 여러 런을 살아남으면 **"기억"**을 축적 (UI 상 작은 흉터 아이콘, 이름 변화). 예: 그라힘의 파편이 3번째 런에 살아남으면 "불탄 파편" 으로 표시 변화.
  - 모라스 시험 이벤트에서 포기된 카드는 **"잊혀진 자들"** 카테고리로 컬렉션에 남음 (사용 불가, 이름만). 잊힘의 대가가 시각화.
  - 이건 내러티브보다는 **수집 메타의 감정선** 작업. 트랙 5 제안 + 트랙 1 구현.

## 8. Hearthstone / Marvel Snap — 피할 것 위주

### Hearthstone
- 4th wall 깨는 유머, 팝문화·밈 레퍼런스, 메타 농담.
- **우리 디자인 규칙 `05-design-direction.md` 가 명시적으로 금지**. 이 레퍼런스는 **안티 패턴 참고용**.
- 단, 한 가지 훔칠 만한 것: **카드 hover 시 보이스 클립**. 카드마다 고유 짧은 보이스(1~2초). 신 등급에만 적용하면 프로덕션 부담 감당 가능.

### Marvel Snap
- lore 거의 없음. 캐릭터는 마블 기존 IP 의존. **학습할 만한 서사 설계 없음**.
- 단, **카드 아트·보더의 레어리티 시각 차별화**는 극단적으로 잘함. → `05-design-direction.md` B-2 규칙과 상통. 우리는 이미 이 방향.

## 9. 적용 우선순위 (다음 작업 큐)

아래는 본 문서 검토 후 실제로 착수할 만한 작업들. 숫자 = 권장 순서.

### 🔴 P0 — 짧고 임팩트 큰 것
1. **lore-bible.md 에 "이념" 한 단어 추가** — 신별 "시험" 옆에 한 단어 이념 라벨 병기. (Genshin 패턴)
2. **카드 플레이버 텍스트 규칙 문서화** — 본 문서 §5 의 "길이/포맷/금지/권장" 4항목을 `design/card_flavor_rules.md` 로 분리 작성.
3. **divine 6장 플레이버를 데이터에 반영 위임** — 초안 완성. 트랙 1(메인)에 "적용만 해달라" 요청.

### 🟡 P1 — 중간 규모
4. **NPC 대사 시스템 설계안**: 성/선술집 NPC 가 지난 런을 멘션하는 Hades 드립피드 구조. 데이터 스키마 + 초기 대사 20~30 줄.
5. **이벤트 내러티브 템플릿 3종**: 파편vs정화 / 맹세vs계약 / 기억포기 — 각 템플릿에 5~10 개 바리에이션 작성.
6. **여신의 "흔적" 시각 규칙**: 실·베틀·무늬 모티프 가이드. 트랙 1(아트) 과 공유.

### 🟢 P2 — 장기
7. **엔딩 분기 설계**: "순수한 후계자" / "불완전한 후계자" / "거부한 자" 3 엔딩. 플레이어 런 데이터 어느 플래그로 분기할지.
8. **신별 연작 플레이버**: LoR "Adventures with Chip" 처럼 한 신의 bronze~legendary 카드 10장을 읽으면 미니 스토리 완성되는 구조.
9. **카드 개체 "기억" 시스템**: Inscryption 식으로 카드가 런을 살아남으면 시각 변화. 데이터 + UI 구현 부담 있음 → 트랙 1 승인 필요.
