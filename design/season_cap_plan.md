## 1. 시간축 결정

| 시즌 | 시즌 Lv 캡 | 등급 캡 (level_system_2026-04-22) | 기간 |
|---|---|---|---|
| 시즌 1 | **50** | bronze 60 / silver 70 / gold 80 / leg 90 / div 100 | **12 주 (3 달)** |
| 시즌 2 | **60** | (동일) | **12 주 (3 달)** |
| 시즌 3 | **70** | (동일) | **12 주 (3 달)** |
| 시즌 4 | **80** | (동일) | **12 주 (3 달)** |
| 시즌 5 | **100** | (동일) | **12 주 (3 달)** — 이후 시즌 6+ 는 캡 100 유지, 다른 풀 콘텐츠 |

> **시즌 길이 = 3 달 (12 주, 84 일)** — `monetization.md` v2.1 § 0 + § 4.1 확정값 준수. 5 시즌 풀 만렙까지 = **15 개월** (1년 3개월). "5시즌 동반자" 정서가 시간축으로 정확히 매칭.

### 곡선 변경 이력 (2026-05-03)

- **v1 초안 (자율 세션)**: 20→40→60→80→100. 첫 시즌 캡 20 = 첫 시즌 너무 빨리 끝남, 신규 진입자 카타르시스 부족.
- **v2 (사용자 결정)**: **50→60→70→80→100**. 첫 시즌부터 충분히 즐길 수 있는 수준 (50 = bronze 등급 캡 60 의 83%) + 시즌 2~4 는 +10씩 점진적 + 시즌 4→5 가 +20 으로 가장 어려운 마지막 구간 ("5시즌 동반자 카타르시스" 가 시즌 5 에 집중).

### 왜 이 곡선?

1. **첫 시즌 = 짧고 만렙 도달 불가**. 1~2시즌엔 만렙 못 가게 → 카드 키우는 즐거움 5시즌까지 유지.
2. **시즌 캡 ≤ 등급 캡 일 때 시즌 캡이 우선**. bronze Lv 60 카드도 시즌 1 에선 Lv 20 까지만 키울 수 있음.
3. **시즌 5 도달 시 Lv 100 = 만렙 = 신좌 도전 가능**. A안의 점유자 시스템과 정확히 묶임.

### 누적 XP (산식만 — 절대치는 트랙 2 검산 후 갱신)

```
xp_to_next_level(L) = 100 × L^1.5  (기존 04-balance § 경험치 곡선 — 정본)
```

> **주의**: 시즌 구간별 누적치 절대값은 본 문서에서 의도적으로 비워둠. `level_system_2026-04-22.md` § 1 의 등급별 누적치(divine 풀 만렙 ≈ 98 만 XP)와 합산 방식이 일치하도록 트랙 2 (data-balance) 가 검산 후 표기. 두 문서 간 14배 차이 발견 시 산식 동일·합산 단위 차이로 추정 — 검증 필수.

### 2.3 신규 헬퍼

```js
// js/50_game_core.js Game 객체에 추가
getEffectiveLevelCap(card){
  const rarityCap = RARITY_CAP[card.rarity || 'bronze'];   // 60/70/80/90/100
  const seasonCap = this.seasonLevelCap || 20;
  const globalCap = this.CARD_LEVEL_CAP;                   // 100
  return Math.min(rarityCap, seasonCap, globalCap);
}
```

기존 `giveCardXp` 의 `CARD_LEVEL_CAP` 참조를 `getEffectiveLevelCap(card)` 로 교체.

## 3. 시즌 캡 풀기 메커니즘

```
시즌 N 종료 → 시즌 N+1 시작
  ↓
1) seasons.level_cap 값이 20 → 40 으로 자동 갱신 (DB 데이터)
  ↓
2) 클라이언트가 다음 로그인 시 Game.seasonLevelCap = 40 으로 동기화
  ↓
3) 기존 Lv 20 으로 묶여있던 카드들이 Lv 40 까지 성장 가능 상태로 풀림
   - 자동 레벨업은 안 됨. exp 만 다시 받기 시작
   - 기존 cap 도달 시점 이후 받은 exp 가 누적되어 있다면 즉시 레벨업 (overflow 처리)
  ↓
4) 풀림 알림 (in-game notification + 시스템 메시지 + 채팅)
   "시즌 2 가 시작되었습니다! 카드 레벨 캡이 40 까지 풀렸습니다."
```

### overflow 처리 (사용자 결정 필요)

**옵션 A** (단순 — exp 안 쌓음): cap 도달 시 exp 그냥 버림. 시즌 풀려도 Lv 1 부터 다시 키워야.
**옵션 B** (저장 — exp 쌓음): cap 도달 시 exp 누적 보관. 시즌 풀리면 한 번에 레벨업.
**옵션 C** (할인 — exp 1/2 쌓음): 절반만 보관.
**옵션 B'** (저장 + 상한): 다음 시즌 cap+10Lv 분량까지만 누적, 초과분은 **신의 잔재**로 환원.

**권장**: **B' (저장 + 상한)** — game-designer P1 결과 반영.
- "5시즌 동반자" 정서 = 시즌 풀리는 순간 폭발적 성장 (B의 카타르시스 유지)
- 그러나 무제한 적립 시 시즌 2 시작 30분 만에 cap 40 도달 가능 → 정서 깨짐 (B의 약점)
- B' 가 절충: 다음 cap 분량 + 10Lv 보너스(약 2주치 단련량) 까지만 적립. 초과분은 신의 잔재로 자동 환원 (단련 가속·시즌 패스 진척에 재투자 가능).
- 상한 도달 시 단련 비활성 + tooltip "이 카드의 운명이 가득 찼습니다" 표시.

## 4. catch-up 메커니즘 (신규 진입자)

시즌 3 이후 합류한 신규 플레이어가 시즌 1~2 의 진척을 빠르게 따라잡을 수 있도록.

### 4.1 누적 단련 골드 할인

- 현재 단련 비용: `100 × L^1.5` (= xp_to_next_level)
- catch-up: **시즌 N 시점에 합류한 신규 플레이어는 Lv (시즌 N-2 의 캡) 까지 단련 비용 50%**
  - 예: 시즌 3 합류 → 시즌 1 캡 Lv 20 까지 비용 50% (시즌 2 캡 40 까지의 절반 구간 = 20)
- 데이터: `seasons` 에 `catchup_until_level` 컬럼 추가 (옵션)

### 4.2 이전 시즌 보상 약식 패키지

- 시즌 N 합류 시 "이전 시즌 보상 패키지" 1회 지급
  - 시즌 1 보상: 골드 5천 + bronze 카드 1장 + 보석 1개
  - 시즌 2 보상: 골드 1만 + silver 카드 1장 + 보석 3개
  - ...
  - 시즌 (N-1) 까지 모두 약식 지급 (정식 시즌 패스 보상의 30~50%)

### 4.3 가속 단련권 (소모품)

- monetization.md § XP 부스터와 통합 검토. "신의 잔재" 사용 시 단련 효율 2배 (1시간 한정 등).
- 무료 지급: 시즌 N 합류 시 가속 단련권 N-1 개.
- **P2W 경계선 명기 (lore-consistency P2-3)**: 가속 단련권 획득 경로가 **무료 한정**(catch-up 패키지·미션·시즌 패스 무료 트랙) 인지, **유료 포함**(보석 → 신의 잔재 환전 → 가속권) 인지 확정 필요. 권장: **유료 환전 가능하되 시즌당 환전 상한** (예: 시즌당 최대 5개) 으로 P2W 임계점 차단.

## 7. MVP 단계 분할

### STEP 1 — 캡 강제 (1일)
- `seasonLevelCap = 20` 글로벌 상수로 박고 `giveCardXp` 가 cap 초과 시 멈춤

- cap 도달 카드 단련 버튼 회색 + tooltip "시즌 2 시작 시 풀림"
- **체감**: 사용자가 "지금 시즌 1 이고 만렙은 5시즌 후" 라는 시간축을 즉시 인지

### STEP 2 — 시즌 전환 (2~3일)
- Supabase `seasons` 테이블 + 운영자가 시즌 종료/시작 시각 수동 갱신
- 클라이언트가 로그인 시 현 시즌 동기화
- 시즌 종료 → 시작 모달 (위 5.2 와이어)
- exp overflow 저장 + 풀림 시 자동 레벨업

### STEP 3 — catch-up (시즌 2 이후)
- 시즌 3 시점에 합류한 신규에게 약식 보상 패키지 + 단련 할인
- monetization.md "가속 단련권" 통합

## 8. A안과의 결합

- A안 점유자 = 시즌 마감 시 1위 카드. **B안 시즌 정의가 없으면 A안 작동 불가**.
- A안 신전 회랑 = 시즌별 점유자 타임라인. B안 시즌 ID 가 그 타임라인의 축.
- → **A+B 동시 phase 강력 권장**. 둘 다 `seasons` 테이블 + 시즌 마감 trigger 를 공유.

## 9. 위험 / 미결 / 사용자 결정 필요

### 9.1 시즌 1 시작 시점
- **옵션 A**: 게임 출시 시점부터 시즌 1 = 곧 출시 = 시즌 1 시작
- **옵션 B**: 출시 후 1 개월간 "시즌 0 베타" + 시즌 1 정식 시작
- **권장 B** — 베타 기간엔 캡 풀린 상태(100)로 운영 + 정식 시작 시 캡 20 으로 리셋 (사용자 카드 Lv 가 20 초과면 그대로 유지 + cap 까지만 노출, 시즌 2 부터 다시 자유)

### 9.2 overflow exp 처리 (§ 3 옵션 A/B/C)
- **권장 B (전부 저장)** — 5시즌 동반자 정서에 부합

### 9.3 신규 진입 시즌 5 합류자 (game-designer P1 개정)
- 시즌 5 합류자가 1주일 만에 Lv 100 도달 가능?
- **권장 개정**: catch-up 은 **Lv 90 까지만 단축**. Lv 91~100 = 시간 게이트 (단련 + 누적 플레이 시간 200시간 또는 다음 시즌 진입 시).
- 신좌·정점 호칭 도전 자격은 `throne_system_plan.md` § 9.0 의 3중 게이트 적용 — **합류 시즌엔 후보 자격 없음, favorite 카드 시즌 카운트 ≥ 3 필요**.
- 효과: 1주일 만렙은 가능하지만 "정점의 신" 호칭은 5시즌 헌신해야만 가능 → 정체성 "5시즌 동반자" 정서 보존.

### 9.4 시즌 6 이후
- 시즌 5 = 만렙 = 끝? 그 후엔?
- **권장**: 시즌 6+ 는 캡 100 유지 + **다른 풀 콘텐츠** (전직 트리·장비·새 신좌 도전 등). 만렙 도달이 끝이 아니라 새 게임의 시작.

### 9.5 카드 레벨 캡 vs 영웅 레벨 캡
- 본 문서는 카드(유닛) 레벨. 영웅(`isHero`) 도 같은 캡인가?
- **권장**: **영웅도 동일 캡 적용** — 영웅이 시즌 1 만에 만렙 가면 정체성 ② 육성 단계가 너무 빨리 끝남. 같은 곡선.
