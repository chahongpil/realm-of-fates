🎯 기본 조작

  • 드래그 = zone 이동
  • 우하단 흰 핸들 = 리사이즈
  • Shift = 스냅 해제
  • 툴바 드롭다운 = 19화면 전환
  • 🔄 iframe 재로드 = 게임 미리보기 새로고침
  • 🎯 실제 DOM 에 맞춤 = 진짜 요소 rect 에 zone 강제 맞춤

📝 텍스트 편집 (stage 아래)

  • 각 행의 textarea 에서 텍스트 수정 → 💾 텍스트 저장
  • css/text_overrides.json 에 기록됨
  • 게임 런타임에 textContent 로 덮어씀 — index.html 수정 안 함
  • 빈 칸 = 원본 복원

🗑️ 숨김 토글

  • 각 행 우측 🗑️ 버튼 → display:none 처리
  • css/hidden_elements.json 에 셀렉터 배열 저장
  • 같은 버튼 다시 누르면 ↩︎ 복구
  • 나중에 AI 가 해당 요소를 index.html 에서 물리 삭제

➕ 지시 추가 모드 (디자인 핸드오프)

  • 툴바 ➕ 지시 추가 클릭 → 커서가 십자로
  • 스테이지를 드래그해서 네모 그리기
  • 릴리즈 시 모달 → 한국어로 "여기에 뭐 넣어줘" 입력
  • 더블클릭으로 편집, × 로 삭제, 드래그로 이동
  • ESC = 모드/모달 취소
  • 저장 파일: css/annotations.json
  • 다음 세션에 AI 가 모든 지시를 읽고 자동 반영
💡 권장 지시 템플릿
"이 영역에 hud_bar 배너 배경으로 배치. 높이 44px, 좌우 20px 패딩. 클릭 시 메뉴 토글."

🧩 Zone 구조 개념

  • 스테이지 = 1280×720 고정 (16:9 HD)
  • 대부분 zone 은 스테이지 절대좌표
  • 일부(auth-box 자식 등)는 부모 rect 상대좌표 — 자동 계산됨
  • 저장 = 10_tokens.css 의 LAYOUT_VARS 블록 갱신
